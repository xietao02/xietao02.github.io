C++11 新特性：列表初始化

在C++11及以后的版本中，列表初始化（list initialization）是一种通用的初始化方式，它可以用于多种不同的场合。

列表初始化可以分为两种类型：直接列表初始化（direct-list-initialization）和复制列表初始化（copy-list-initialization），它们的区别主要体现在是否允许调用显式构造函数上。

直接列表初始化 (Direct-list-initialization)

直接列表初始化使用花括号 {} 而不是等号 = 。它可以用来初始化任何类型的对象，并且在初始化时会考虑对象的显式和非显式构造函数。

显式和非显式构造函数

在C++中，构造函数可以被声明为 explicit ，这意味着不能通过隐式类型转换来调用该构造函数。只有当直接调用构造函数或使用直接列表初始化时，才能使用显式构造函数。

  • 显式构造函数（Explicit constructor）：通过在构造函数声明前添加 explicit 关键字来定义。这样的构造函数阻止了编译器自动使用该构造函数进行隐式类型转换。

  • 非显式构造函数（Non-explicit constructor）：默认情况下，构造函数是非显式的，允许隐式类型转换。

何时使用列表初始化？

直接列表初始化 (direct-list-initialization)

  1. 命名变量的初始化：使用花括号初始化一个命名变量，例如 int x{5}; 。
  2. 未命名临时对象的初始化：使用花括号初始化一个未命名的临时对象，如在函数调用中直接传递初始化列表。
  3. 具有动态存储持续时间的对象的初始化：使用 new 表达式和花括号初始化列表来创建动态分配的对象，例如 auto ptr = new std::vector{1, 2, 3}; 。
  4. 非静态数据成员的就地初始化：在类定义内部使用花括号而不是等号初始化成员变量。
  5. 构造函数的成员初始化列表：如果使用花括号初始化列表，则在构造函数的成员初始化列表中进行列表初始化。

复制列表初始化 (copy-list-initialization)

  1. 命名变量的初始化：在等号后使用花括号初始化列表初始化一个命名变量，例如 int x = {5}; 。
  2. 函数调用中的参数初始化：函数调用中使用花括号初始化列表作为参数，对函数参数进行列表初始化。
  3. return语句中的对象初始化：在return语句中使用花括号初始化列表返回对象。
  4. **下标表达式中的用户定义的operator[]**：在用户定义的下标操作符中，使用列表初始化来初始化重载操作符的参数。
  5. 赋值表达式中的列表初始化：在赋值表达式中，使用列表初始化来初始化重载赋值操作符的参数。
  6. 函数式转换表达式或其他构造函数调用：使用花括号初始化列表替代构造函数的参数。
  7. 使用等号的非静态数据成员初始化：在类定义中使用等号和花括号初始化列表初始化非静态数据成员。

列表初始化为C++提供了一种统一且灵活的初始化语法，它可以用于多种不同的场景，而不再需要单独区分变量或对象，并且有助于防止窄化转换等潜在问题，提高了代码的健壮性和可读性。

注意：为了使用C++11的特性，需要在编译命令中添加 -std=c++11 或更高版本的标准
